# TRANSITIONS

Essentially every transition is a function that allows you to switch pattern in a certain way. All of them need at least one argument: ID of the pattern and time in which the transition will be performed, if its an `In` version of it.

## Anticipate
| Name | Description | Example |
| ---- | ----------- | ------- |
|`anticipte`| Drop in `8` cycles | `anticipate 1 $ s "bd sn"` |
|`anticipteIn`| Drop in `n` cycles | `anticipate 1 4 $ s "bd sn"` |

## Clutch
| Name | Description | Example |
| ---- | ----------- | ------- |
|`clutch`| Degrades the current pattern while undegrading the next in `2` cycles| `clutch 1 $ s "bd sn"` |
|`clutchIn`| Degrades the current pattern while undegrading the next in `n` cycles| `clutchIn 1 4 $ s "bd sn"` |

## History paning
| Name | Description | Example |
| ---- | ----------- | ------- |
|`histpan` | Pan last `n` patterns, recent to the left | `histpan 1 4 $ sound "bd sn"` |

## Interpolation
| Name | Description | Example |
| ---- | ----------- | ------- |
|`interpolate`| Linear interpolation between CVs in `4` cycles | `interpolate 1 $ s "bd sn" #cutoff 100`|
|`interpolateIn`| Linear interpolation between CVs in `n` cycles | `interpolateIn 1 4 $ s "bd sn" #cutoff 100`|

## Jumps
| Name | Description | Example |
| ---- | ----------- | ------- |
|`jump`| Jump to a new pattern | `jump 1 $ s "bd sn"`|
|`jumpIn`| Jump to a new pattern in `n` cycles unaligned to the main cycle | `jumpIn 1 4 $ s "bd sn" `|
|`jumpIn`| Jump to a new pattern in `n` cycles aligned to the main cycle | `jumpIn' 1 4 $ s "bd sn" `|
|`jumpMod`| Jump to a new pattern in `n` cycles aligned to the main cycle | `jumpMod 1 4 $ s "bd sn"`|

## Wait
| Name | Description | Example |
| ---- | ----------- | ------- |
|`wait`| Pause the current pattern and then play a new one |  `wait 1 4 $ s "bd sn"`|
|`waitT` | Allows to use any of the pattern transitions| `waitT 2 (Sound.Tidal.Transition.xfadeIn 2) 4 $ s "hh*8"`|

## Fades
| Name | Description | Example |
| ---- | ----------- | ------- |
| `xfade` | Crossfade to a new pattern | `xfade 1 $ s "arpy*8" # n (run 8)` |
| `xfadeIn` | Crossfade to a new pattern in `n` cycles | `xfadeIn 1 4 $ s "arpy*8" # n (run 8)` |
| `fadeIn` | Fade in to a new pattern in `n` cycles | `fadeIn 1 4 $ s "arpy*8" # n (run 8)` |
| `fadeInFrom` | Fade in to a new pattern in `n` cycles after `m` cycles| `fadeInFrom 1 4 4 $ s "arpy*8" # n (run 8)` |
| `fadeOut` | Fade out to a new pattern in `n` cycles | `fadeOut 1 4 $ s "arpy*8" # n (run 8)` |
| `fadeOutFrom` | Fade out to a new pattern in `n` cycles after `m` cycles| `fadeOutFrom 1 4 4 $ s "arpy*8" # n (run 8)` |
